# 04-02-2026

Все перечисленные справочники и данные находятся в подсистеме "Добавленые объекты"  

1. Создайте группу и бота в Телеграмм. Они нужны для отправки уведомлений специалистам о созданных документах «Обслуживание клиента». Уведомления создаются при создании нового документа, а так же при изменении даты и ФИО специалиста.

3. Заполните следующие константы: "Сервис" «Номенклатура абонентская плата», при необходимости создайте нужную номенклатуру; "Сервис" «Номенклатура работы специалиста» —  при необходимости создайте нужную номенклатуру; «Токен управления телеграм ботом» — заполняется токеном бота телеграмм; «Идентификатор группы для оповещения» — заполняется идентификатором группы телеграмм.

4. Заполните данные об окладах сотрудников в "Расчеты" «Условия оплаты сотрудников», укажите процент от работ у Специалистов и оклады у всех сотрудников.

5. В справочник «Графики работ (Справочник)» заведите необходимый вид графика работы, например пятидневка. Затем в подсистеме «Сервис» выберите обработку «Заполнение графика работы». В поле выбор периода выберите период (год) для заполнения графика. В поле «Выходные дни» через запятую перечислите порядковый номер выходного дня недели. Например: 6,7 (как суббота и воскресенье). В поле «график работы» выберите ранее созданный элемент справочника «Графики работы». Нажмите кнопку «Заполнить». Далее в регистре «График работы» проверьте корректность заполнения графика. Будьте внимательны, для расчетов с использованием календарных дней, необходимо отдельно создать и заполнить график «Календарные дни».

6. В Администрирование - Настройки пользователей и прав - Польхзователи - создайте пользователей. Например: Специалист Специалист А.А. как пользователь «Специалист А.А.». На вкладке права доступа назначьте каждому пользователю соответствующий должности профиль группы доступа. Наименования профиля группы доступа созданы для вашего удобства интуитивно понятными. Пример: для бухгалтеров профиль группы доступа «БухгалтерИТФирмы», для кадровика «Кадровик расчетчик». Все созданные профили групп доступа являются поставляемыми и не могут быть изменены пользователем самостоятельно. Подробнее с составом ролей профилей групп доступа вы можете ознакомится по следующему пути: «Администрирование» -> «Настройки пользователей и прав» -> «Профили групп доступа».

7. Проинструктируйте бухгалтера о возможности автоматического формирования документов по реализации за месяц с помощью обработки "Серис" «Создание актов». Период в обработке должен быть выбран предыдущий месяц.
8. Проинструктируйте менеджера о возможности создания договоров вида Абонентское обслуживание. Для них реализовано заполнение дополнительных полей: «дата начала» и «окончания договора», «стоимость часа работы», «Абонентская плата». Эти поля должны быть заполнены для корректного формирования документов реализации.
9. Так же проинформируйте пользователя о том, что при не заполненных константах «Работы специалиста» и «Абонентская плата» документ заполнен не будет.
10. Так же создан «График отпусков» в "Отпуска" позволяющий планировать отпуска специалистов. При превышении 28 дней отпуска у работника в таблице строки с данными работника выделяются цветом. Также по кнопке "Показать диаграмму" выводится наглядный крафик отпусков с данными работников и их отпусками.
11. Добавлены отчеты "Выработка специалистов", "Анализ выставленых актов", "График отпусков", "Расчеты с сотрудниками". Отчеты интуитивно понятны выдают информацию в соответствии с названиями и задаваемыми периодами.
